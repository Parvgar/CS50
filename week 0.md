# Week 0 - CS50

## What I Learned
- Basics of programming logic using Scratch
- How events work (when green flag clicked, when I receive)
- Conditions using if blocks
- Variables like score and rounds
- Basic idea of game flow (input → action → result)
- How different sprites interact in a project

## What I Worked On (Project)
I tried building a simple Scratch-based football penalty game idea where:
- A referee starts the game
- Player selects number of rounds
- Ball is kicked using keyboard input
- Goal or save is detected using conditions
- Score is updated based on result

## Challenges I Faced
- Confusion in finding Scratch blocks
- Difficulty connecting different sprites using broadcast
- Understanding how events and conditions work together
- Initially tried building a complex system which felt overwhelming

## How I Overcame It
- Broke the problem into smaller parts
- Focused on one feature at a time (like only ball movement first)
- Understood that Scratch works with separate scripts and events
- Simplified the project instead of making it too complex

## Outcome
- Learned how to think step-by-step like a programmer
- Understood event-driven programming basics
- Gained confidence in building simple interactive logic
- Completed a working understanding of a Scratch-based game system
